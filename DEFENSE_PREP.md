# 🛡 App Defense Preparation Guide

Use this guide to prepare for your presentation. It highlights the technical decisions and architectural strengths of the application.

## 🎤 1. High-Level Summary
- **Pitch**: "TaskFlow is an offline-first task manager designed for productivity. It doesn't just list tasks; it actively guides the user through Focus Mode and tracks long-term consistency using building habits (the streak heatmap)."
- **Tech Stack**: Flutter (Frontend), Firebase (Backend/Auth), Riverpod (State Management).

## 🧩 2. Technical Walkthrough (The "Meat")

### **Architecture: Why Feature-First?**
*   "I chose a feature-first structure (`auth`, `tasks`, `profile`) because it makes the app modular. If I want to add a 'Calendar' feature, I can drop it in as a new folder without touching the rest of the app's logic."
*   **Keywords**: Scalability, Separation of Concerns, Modularity.

### **State Management: Riverpod**
*   "I used Riverpod because it is compile-safe and doesn't rely on the BuildContext as much as Provider. This allowed me to keep business logic completely out of the UI."
*   **The 'Cache-then-Network' Pattern**: "In the `TasksScreen`, I use `StreamProvider`. It pulls cached data immediately from Firestore and then updates in real-time when the network responds. This prevents the user from seeing blank screens."

### **Navigation & Security**
*   "I implemented `GoRouter` with nested shell routes. The `AppRouter` has a redirect logic that checks your `authState`. If you aren't logged in, you can't access the home screen—period."

### **REST API & Dio**
*   "I used `Dio` to fetch the 'Daily Advice'. I implemented error handling so that if the API is down or the user is offline, the app shows a fallback message instead of crashing."

## ❓ 3. Likely Questions & Answers

**Q: Why did you use Firestore offline persistence?**
> "It's critical for a task manager. Users need to add or check tasks in subways or areas with bad signals. My app queues the writes and syncs them automatically when they reach 4G/WiFi."

**Q: How did you implement the streak map?**
> "I used the `flutter_heatmap_calendar` package. I feed it a dataset generated by mapping through professional Firestore task objects and grouping them by their `createdAt` date."

**Q: How do you handle secrets?**
> "Sensitive data isn't hardcoded. I use `firebase_options.dart` (which should be in .gitignore in a production environment) and secure storage for sensitive tokens."

**Q: Show me the Domain Layer.**
> "Open `lib/features/tasks/domain/entities/task.dart`. This is a pure Dart class with no external dependencies. It defines exactly what a Task is for our system, independent of whether we use Firestore or a SQL database."
